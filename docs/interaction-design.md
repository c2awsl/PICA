# PICA 交互设计重设计 — 可视化的分类/标签系统

> 目标：解决「待确认页面让人不知所措」的问题，让分类/标签完全可视化、可浏览、可操作。

---

## 一、核心问题诊断

### 当前待办页的问题

```
❌ 9 种视图模式 → 选择瘫痪
❌ 所有图片平铺展示 → 信息过载
❌ 分类/标签只是小字 → 不可视
❌ AI 做了什么不可见 → 黑箱感
❌ 操作入口隐蔽 → 不知道点哪里
❌ 没有浏览路径 → 只能「全部处理」
```

### 用户心理模型

用户真正想做的是：

```
1. "看看 AI 把我的图片分成了哪些类"
2. "嗯，原神这批分类很准，全部确认"
3. "有几张梗图分错了，改一下"
4. "这组相似的挑一张留下"
```

**而不是**：在一大屏缩略图里挨个点 hover 按钮。

---

## 二、核心理念：Browse-First（先浏览，再处理）

### 把待办页从「工作台」变成「画廊」

```
当前:   筛选 → 浏览 → 操作（扁平列表，认知负荷高）
重新设计: 浏览 → 聚焦 → 操作（树形浏览，逐步缩小范围）
```

### 三态浏览模式

用户通过三个递进的步骤处理图片：

```
Step 1: 浏览分类/标签
        ↓ 选择感兴趣的分类
Step 2: 查看该分类下的图片
        ↓ 逐张或批量操作
Step 3: 确认/驳回/编辑
```

## 四、交互流程设计

### 4.1 默认用户路径（新手引导）

```
首次打开待确认页:
  ↓
「你有 45 张新图片需要处理」
  ↓
显示分类概览（大卡片）:
  [梗图 23张] [原神 12张] [截图 10张]
  ↓
用户点击「梗图」
  ↓
进入梗图分类详情:
  - 顶部: "梗图 · 23 张待确认"
  - 左边: 分类树（突出当前选中梗图）
  - 右边: 图片网格
  ↓
用户悬停/点击图片:
  - 卡片展开显示 AI 建议 + 操作按钮
  ↓
操作:
  ✓ 确认（使用 AI 建议 + 可选修改）
  ✕ 驳回
  ✏ 编辑分类/标签
```

### 4.2 批量操作流程

```
在分类详情页:
  [✓ 全部确认] [批量编辑] [批量驳回]

点击[✓ 全部确认]:
  → 弹窗: "将 23 张梗图全部归档？"
  → 确认后: 全部归档到「梗图」分类，保留 AI 标签
  
点击[批量编辑]:
  → 进入选择模式（勾选需要的图）
  → 底部出现操作栏:
     [添加标签] [移除标签] [改分类] [改作品] [确认] [驳回]
```

### 4.3 单张操作流程

```

卡片操作区（始终可见）:

┌──────────────────────────────┐
│ 📷 缩略图                      │
│ filename.png                  │
│                               │
│ AI 建议: [梗图] [原神] [对话]  │
│ 置信度: ⭐⭐⭐⭐                │
│                               │
│ ┌──────────────┐  ┌────────┐  │
│ │ 分类: 梗图 [▼]│  │ ✓ 确认 │  │
│ │ 作品: 原神 [▼]│  │ ✕ 驳回 │  │
│ │ 标签: [✏编辑] │  │        │  │
│ └──────────────┘  └────────┘  │
└──────────────────────────────┘

点击 [▼] 选择其他分类/作品 → 即时生效
点击 [✏编辑] 弹出标签编辑器 → 搜索/选择已有标签 → 确认
点击 [✓ 确认] → 用当前分类/标签归档
点击 [✕ 驳回] → 确认 → 移除
```

### 4.4 防呆设计清单

| 场景 | 防呆措施 |
|------|---------|
| 首次使用 | 显示引导提示「点击分类开始浏览」 |
| 空状态 | 显示「暂无图片，同步到 sync_dir 后会自动识别」 |
| 批量确认 | 弹窗确认「将 XX 张归档？不可撤回」 |
| 驳回 | 软删除（标记状态，不删文件），30 天内可恢复 |
| 分类/标签选择 | 先显示 AI 建议（高亮），用户点击即选中 |
| 修改分类后 | 自动更新归档路径 |
| 标签合并 | 二次确认「合并后源标签将被移除」 |
| AI 失败 | 显示错误原因 + 重试按钮 |
| 操作太快 | 确认按钮有 300ms 防抖 |

## 五、视觉效果规范

### 5.1 卡片设计

## 七、关键设计决策

1. **Browse-First 替代 List-First**：默认展示分类概览而非图片列表，降低认知负荷
2. **三态模式**：概览 → 分类详情 → 单张操作，递进式交互
3. **分类即导航**：左边分类树 = 导航 + 状态面板，不用额外页面
4. **AI 透明化**：每张卡片显示 AI 建议了什么、置信度多少
5. **操作常驻**：不隐藏、不 hover，降低学习成本
6. **防呆优先**：每个操作都有确认、都有反馈、都可撤销
7. **标签可视化**：标签是彩色可点击的芯片，不是纯文本
